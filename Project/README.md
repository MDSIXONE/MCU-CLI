# MCU CLI 项目目录

## 目录说明

本目录存放所有 MCU CLI 相关的任务项目。每个任务都有独立的文件夹，包含任务文档和相关文件。

## 目录结构

```
Project/
├── README.md                    # 本文件
├── ESP32_S3LED/                 # 示例：ESP32 S3 LED 控制任务
│   └── TASK.md                  # 任务详情文档
├── STM32_Motor/                 # 示例：STM32 电机控制任务
│   └── TASK.md
└── ...                          # 更多任务
```

## 任务文件夹命名规范

- 使用大写字母和下划线
- 格式：`芯片型号_功能描述`
- 示例：`ESP32_S3LED`、`STM32_Motor`、`RP2040_Temperature`

## TASK.md 文档结构

每个任务文件夹必须包含 `TASK.md`，内容包括：

1. **任务概述** - 任务目标和背景
2. **硬件信息** - MCU 型号、引脚连接、外设配置
3. **完成的工作** - 已实现的功能列表
4. **遇到的问题** - 问题描述
5. **解决方案** - 如何解决问题
6. **测试结果** - 测试通过情况
7. **下一步计划** - 后续工作
8. **相关文件** - 代码、配置、文档链接

## 使用方法

### 创建新任务
```bash
# 1. 创建任务文件夹
mkdir Project/ESP32_S3LED

# 2. 创建 TASK.md
# 按照模板填写任务详情
```

### 更新任务
每次完成任务后，更新 TASK.md 中的：
- 完成的工作
- 遇到的问题
- 解决方案
- 测试结果

### 查看任务
```bash
# 查看所有任务
ls Project/

# 查看特定任务
cat Project/ESP32_S3LED/TASK.md
```

## 注意事项

1. **及时更新** - 每次任务完成后立即更新 TASK.md
2. **详细记录** - 记录问题和解决方案，方便后续参考
3. **保持整洁** - 删除临时文件，只保留必要文件
4. **版本控制** - 重要任务建议使用 git 管理
